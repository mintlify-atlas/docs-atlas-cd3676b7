## Overview

The Crepes & Waffles Coworking system uses GPS-based geolocation to verify user attendance at the coworking space. The `geolocationService` provides functions for:

- Calculating distance between coordinates (Haversine formula)
- Verifying users are within the allowed radius
- Managing geolocation permissions
- Time-window based verification
- Cross-platform browser compatibility

<Note>
  Geolocation verification requires HTTPS in production. The Geolocation API only works on secure origins (HTTPS or localhost).
</Note>

## Distance Calculation

### Haversine Formula

The system uses the Haversine formula to calculate great-circle distance between two GPS coordinates:

```javascript
export const calculateDistance = (lat1, lon1, lat2, lon2) => {
  const R = 6371e3; // Earth radius in meters
  const φ1 = (lat1 * Math.PI) / 180;
  const φ2 = (lat2 * Math.PI) / 180;
  const Δφ = ((lat2 - lat1) * Math.PI) / 180;
  const Δλ = ((lon2 - lon1) * Math.PI) / 180;

  const a =
    Math.sin(Δφ / 2) * Math.sin(Δφ / 2) +
    Math.cos(φ1) * Math.cos(φ2) * Math.sin(Δλ / 2) * Math.sin(Δλ / 2);
  
  const c = 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1 - a));
  return R * c; // Distance in meters
};
```

**Accuracy:** Within ~0.5% for most distances (suitable for geofencing)

## Geolocation Permissions

### Permission States

The Geolocation API has three permission states:

| State | Meaning | User Action |
|-------|---------|-------------|
| `granted` | User allowed location access | Can use GPS immediately |
| `denied` | User blocked location access | Must enable in browser settings |
| `prompt` | User hasn't decided yet | Will show permission dialog |
| `unsupported` | Browser doesn't support API | Use alternative method |

### Checking Permissions

```javascript
import { checkLocationPermission } from '../utils/geolocationService';

const status = await checkLocationPermission();

switch (status) {
  case 'granted':
    console.log('✅ Permission granted');
    break;
  case 'denied':
    console.log('❌ Permission denied');
    break;
  case 'prompt':
    console.log('⚠️ User needs to allow');
    break;
  case 'unsupported':
    console.log('ℹ️ API not available (iOS Safari)');
    break;
}
```

## Position Options

### GPS Configuration

```javascript
const options = {
  enableHighAccuracy: true,  // Use GPS if available
  timeout: 15000,            // 15 seconds timeout
  maximumAge: 5000           // Accept 5-second old positions
};
```

<ParamField path="enableHighAccuracy" type="boolean" default={true}>
  Request high-accuracy GPS position (uses more battery).
  
  - `true` - Use GPS satellite (higher accuracy, slower, more battery)
  - `false` - Use WiFi/cell towers (lower accuracy, faster, less battery)
</ParamField>

<ParamField path="timeout" type="number" default={15000}>
  Maximum time (ms) to wait for position.
  
  **Default:** 15000 (15 seconds)
  
  Increased from standard 5s to support slower devices.
</ParamField>

<ParamField path="maximumAge" type="number" default={5000}>
  Accept cached positions up to this age (ms).
  
  **Default:** 5000 (5 seconds)
  
  Prevents requesting GPS for every check.
</ParamField>

## Attendance Verification

### Complete Verification Flow

```javascript
import { verifyAttendance } from '../utils/geolocationService';

const handleConfirm = async (reserva) => {
  try {
    const result = await verifyAttendance(reserva, {
      horarios: horariosData,      // Optional: shift schedules
      referenceDate: new Date(),   // Optional: for testing
      workplaceInfo: customCoords, // Optional: override defaults
      position: cachedPosition     // Optional: use cached GPS
    });
    
    if (result.success && result.shouldUpdate) {
      // Update reservation status
      await updateReservationStatus(reserva.id, {
        estado: result.newStatus,
        confirmada: result.confirmed,
        verificacionAsistencia: {
          tipo: 'gps',
          distancia: result.distance,
          timestamp: Date.now(),
          posicion: result.position
        }
      });
      
      alert(result.message);
    } else {
      // Show error or warning
      alert(result.message);
    }
  } catch (error) {
    console.error('Verification failed:', error);
  }
};
```

### Verification Result Object

<ParamField path="result.success" type="boolean">
  Whether verification succeeded.
</ParamField>

<ParamField path="result.shouldUpdate" type="boolean">
  Whether reservation status should be updated in database.
</ParamField>

<ParamField path="result.newStatus" type="string">
  New reservation status: `'Confirmada'`, `'Cancelada'`, or `'Pendiente'`.
</ParamField>

<ParamField path="result.withinPerimeter" type="boolean">
  Whether user is within allowed radius.
</ParamField>

<ParamField path="result.distance" type="number">
  Distance in meters from workplace.
</ParamField>

<ParamField path="result.message" type="string">
  Human-readable message for UI display.
</ParamField>

<ParamField path="result.timeInfo" type="object">
  Verification window details:
  - `isToday` - Reservation is for today
  - `isActive` - Within verification window
  - `isPast` - Verification window expired
  - `startTime` - Window start time (e.g., "08:00")
  - `endTime` - Window end time (e.g., "08:25")
</ParamField>

## Error Handling

### Error Codes

<ParamField path="PERMISSION_DENIED" type="1">
  User denied location permission.
  
  **Solution:** Show browser settings instructions.
</ParamField>

<ParamField path="POSITION_UNAVAILABLE" type="2">
  Device cannot determine location.
  
  **Causes:** No GPS, WiFi off, poor signal
  
  **Solution:** Ask user to enable GPS/WiFi.
</ParamField>

<ParamField path="TIMEOUT" type="3">
  Location request timed out.
  
  **Solution:** Retry with higher timeout or lower accuracy.
</ParamField>

## Security Considerations

<Warning>
  **Privacy:** Always inform users why location access is needed. Only request when necessary.
  
  **HTTPS:** Geolocation API only works on secure origins (HTTPS or localhost).
  
  **Accuracy:** GPS can be spoofed. Don't rely solely on location for critical security decisions.
</Warning>

### Best Practices

1. **User consent:** Show clear explanation before requesting permission
2. **Fallback options:** Provide manual check-in alternative
3. **Error handling:** Graceful degradation when GPS unavailable
4. **Privacy policy:** Document how location data is used and stored
5. **Data retention:** Don't store GPS coordinates longer than necessary
