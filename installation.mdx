## System Requirements

Before installing, ensure your environment meets these requirements:

- **Node.js**: v16.x or higher
- **npm**: v7.x or higher (comes with Node.js)
- **Browser**: Modern browser with WebSocket support (Chrome 87+, Firefox 82+, Safari 14+, Edge 88+)
- **HTTPS**: Required for geolocation features in production

### Configure Environment Variables

Create a `.env` file in the project root to enable real-time synchronization:

```bash .env
# WebSocket URL for real-time reservation updates
# If not set, app falls back to local events + polling
VITE_RESERVAS_WS_URL=wss://tu-servidor/ws/reservas

# Polling interval in milliseconds (default: 30000)
# Used as fallback when WebSocket is unavailable
VITE_RESERVAS_POLLING_MS=30000
```

<AccordionGroup>
  <Accordion title="VITE_RESERVAS_WS_URL" icon="plug">
    **Purpose**: WebSocket endpoint for real-time reservation synchronization
    
    **Default**: Not set (uses local events + polling only)
    
    **Format**: `wss://your-server/ws/reservas`
    
    **Usage in code**:
    ```jsx src/hooks/useRealtimeSync.js
    const WS_URL = import.meta.env.VITE_RESERVAS_WS_URL || '';
    
    if (!WS_URL) {
      console.log('ℹ️ WebSocket no configurado. Se usa eventos locales + polling.');
      return undefined;
    }
    
    const ws = new WebSocket(WS_URL);
    ws.onmessage = (event) => {
      const payload = JSON.parse(event.data);
      if (shouldSyncFromSocketPayload(payload)) {
        triggerSync('websocket-message');
      }
    };
    ```
    
    <Warning>
    WebSocket connections require `wss://` (secure) in production. Use `ws://` only for local development.
    </Warning>
  </Accordion>
  
  <Accordion title="VITE_RESERVAS_POLLING_MS" icon="clock">
    **Purpose**: Backup polling interval when WebSocket is unavailable
    
    **Default**: `30000` (30 seconds)
    
    **Format**: Number in milliseconds
    
    **Usage in code**:
    ```jsx src/hooks/useRealtimeSync.js
    const POLLING_MS = Number(import.meta.env.VITE_RESERVAS_POLLING_MS || 30000);
    
    useEffect(() => {
      syncIntervalRef.current = setInterval(() => {
        const timeSinceLastSync = Date.now() - lastSyncRef.current;
        
        if (timeSinceLastSync > POLLING_MS - 5000) {
          console.log('⏰ Sincronización automática (polling)');
          triggerSync('polling');
        }
      }, POLLING_MS);
    }, [triggerSync]);
    ```
    
    <Note>
    Polling acts as a fallback mechanism. For production, configure WebSocket for instant updates.
    </Note>
  </Accordion>
</AccordionGroup>

  <Accordion title="useRealtimeSync.js" icon="arrows-rotate">
    Real-time synchronization hook:
    ```jsx src/hooks/useRealtimeSync.js
    const WS_URL = import.meta.env.VITE_RESERVAS_WS_URL || '';
    const POLLING_MS = Number(import.meta.env.VITE_RESERVAS_POLLING_MS || 30000);
    
    const useRealtimeSync = (onSync) => {
      // WebSocket connection
      useEffect(() => {
        if (!WS_URL) return undefined;
        
        const ws = new WebSocket(WS_URL);
        
        ws.onmessage = (event) => {
          const payload = JSON.parse(event.data);
          if (shouldSyncFromSocketPayload(payload)) {
            triggerSync('websocket-message');
          }
        };
      }, []);
      
      // Polling fallback
      useEffect(() => {
        const interval = setInterval(() => {
          triggerSync('polling');
        }, POLLING_MS);
        
        return () => clearInterval(interval);
      }, []);
    };
    ```

      const { workplaceInfo, referenceDate = new Date() } = options;
      
      // Check time window (first 25 minutes)
      const timeInfo = getVerificationTimeInfo(reserva, [], referenceDate);
      if (!timeInfo.isActive) {
        return { success: false, message: timeInfo.message };
      }
      
      // Check location
      const position = await getCurrentPosition();
      const distance = calculateDistance(
        position.latitude,
        position.longitude,
        workplaceInfo.latitude,
        workplaceInfo.longitude
      );
      
      if (distance > workplaceInfo.radius) {
        return {
          success: false,
          message: `Estás a ${Math.round(distance)}m. Debes estar dentro de ${workplaceInfo.radius}m.`
        };
      }
      
      return { success: true, message: 'Verificación exitosa' };
    };

### Admin Users

Admin users are configured with specific document numbers:

```jsx src/assets/componentes/Panel.jsx
const ADMINS = ['1028783377', '1019096266'];

const esAdmin = ADMINS.includes(documentoUsuario);
```

Admins can:
- View all reservations (not just their own)
- Manually confirm any pending reservation
- Bypass the 25-minute confirmation window
- Reactivate cancelled reservations

## Troubleshooting

<AccordionGroup>
  <Accordion title="npm install fails">
    **Solution**:
    1. Clear npm cache: `npm cache clean --force`
    2. Delete `node_modules` and `package-lock.json`
    3. Run `npm install` again
    4. Ensure Node.js version is 16.x or higher
  </Accordion>
  
  <Accordion title="Port 5173 already in use">
    **Solution**:
    ```bash
    # Use a different port
    vite --port 3000
    
    # Or find and kill the process
    lsof -ti:5173 | xargs kill
    ```
  </Accordion>
  
  <Accordion title="WebSocket connection failed">
    **Solution**:
    1. Verify `VITE_RESERVAS_WS_URL` is correctly set
    2. Ensure WebSocket server is running
    3. Check firewall/network settings
    4. App will fall back to polling if WebSocket fails
  </Accordion>
  
  <Accordion title="Geolocation not working in dev">
    **Solution**:
    1. Use `localhost` (not `127.0.0.1` or IP address)
    2. Enable location permissions in browser
    3. For LAN testing, use HTTPS with self-signed cert
  </Accordion>
</AccordionGroup>
